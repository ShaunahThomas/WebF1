Week 7

Separating Form and Content

Lecture:

In this lecture, the focus was the separation of form and content. It also identified the difference between descriptive and procedural mark-up. The lecture observed that descriptive mark-up is preferable for several reasons, these include the following:

  • ease of maintenance
  • smaller overall file sizes
  • increased semantic encoding, leading eventually to
  • more content that can be processed on our behalf by machines.

The lecture also gave an induction into Cascading Style Sheets (CSS) and why it’s important to use.

Practical:

Task 0: Preparation – In this task I created a folder called style. Within the folder, I also created a html document that contained the example code that I was given. I have never made a stylesheet before, so this was new to me.

Task 1: An Embedded Stylesheet – In this task I created a stylesheet.

Task 2: A Linked Stylesheet – The task was to link the stylesheet I made in task 1 to the html document that I created in task 0.

Task 3: Class & ID Selectors – In this task I added more elements to the style sheet. Those elements include class selectors and id selectors. Examples:

Class Selectors:

In mystyle.css:

.First {

color: red;

}

.Second {

background: black;

}

In linked.html:

<p class="First Second"> Hello </p>

Id Selectors:

In mystyle.css:

#Section1 {

background: blue;

}

In linked.html:

<p id="Section1 " >Hello</p >

Task 4: Read, Discover, Experiment, Learn – This task insisted on reading up more on CSS to get a better understanding of the things you can do with it.
